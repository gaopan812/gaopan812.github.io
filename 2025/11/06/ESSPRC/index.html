资源受限的基本最短路问题

发表于 2025-11-06 更新于 2026-07-04 分类于 运筹优化 阅读次数：

在使用列生成算法求解带时间窗限制的车辆路径规划问题时，定价子问题是一个资源受限的基本最短路问题（Elementary Shortest Path Problem with Resource Constraints，ESSPRC）。在文献中[^1]证明了ESSPRC问题是强NPNPNP难的，因此针对大规模的问题，使用求解器求解速度较慢。

问题描述

有一辆容量为QQQ的货车，从起点ooo出发，为途中的客户送货，最后到达终点ddd， 货车在配送时需要满足时间和容量的限制。假设有mmm个客户，这些客户分布在不同的地点, 客户用集合CCC表示，C={1,2,…,m}C = \{1, 2, \ldots, m\}C={1,2,…,m}。每个客户iii有不同的服务时间窗[ai,bi][a_i, b_i][ai​,bi​]和需求量qiq_iqi​。本问题考虑的是硬时间窗，也就是说给客户iii的服务时间必须落在时间窗 [ai,bi][a_i, b_i][ai​,bi​]内。将起点ooo记作0，终点ddd设为m+1m+1m+1，则所有的节点用集合VVV表示，V={0,1,2,…,m+1}V = \{0, 1, 2, \ldots, m+1\}V={0,1,2,…,m+1}。从节点iii到节点jjj的运输时间为tijt_ijti​j, 运输成本为cijc_ijci​j。问题的目标是，在货车容量不超载，每个客户最多被访问一次且服务开始时刻落在相应的服务时间窗内的情况下，最小化运输总成本。

ESPPRC的松弛

由于ESPPRC是强NPNPNP难问题，在求解时通常会松弛问题中的某些约束。其中，应用非常广泛的就是允许路径中存在子回路，即允许客户被多次访问。如此，问题就变成了资源受限最短路问题（SPPRC）。

数学模型

基于前面介绍过的ESPPRC的模型，Brian Kallehauge 等人对SPPRC也给出了如下数学模型：
在模型中定义了一个额外的集合K={1,2,…,∣K∣}K= \{1, 2, \ldots, |K|\}K={1,2,…,∣K∣}，用来表示路径中弧的顺序。由于每个客户都可以多次访问，所以无法确定最后求得的路径中包含几条弧，但是路径所包含的弧的个数的上限∣K∣|K|∣K∣是可以确定的。路径中弧的数量受限于容量资源和时间资源，所以∣K∣=min⁡{⌊Qmin⁡i∈V{qi}⌋,⌊bm+1min⁡(i,j)∈V×V{tij}⌋}|K|=\min\{\lfloor\frac{Q}{\min_{i\in V}\{q_i\}}\rfloor,\lfloor\frac{b_{m+1}}{\min_{(i,j)\in V\times V}\{t_{ij}\}}\rfloor\}∣K∣=min{⌊mini∈V​{qi​}Q​⌋,⌊min(i,j)∈V×V​{tij​}bm+1​​⌋}。
需要注意的是，虽然在SPPRC中，每个客户允许被多次访问，但是，起点和终点在路径中还是只能出现一次。
